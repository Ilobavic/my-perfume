## Design Context

### Users
Single personal user managing a small fragrance collection (~9 bottles, growing). Context: deciding what to wear today based on weather, occasion, and mood; browsing and editing bottle details; tracking rotation and depletion. Used on phone and desktop, often as a quick daily decision tool.

### Brand Personality
Intimate · Sensory · Confident

Emotional goals: calm certainty when picking a scent; quiet luxury without showroom flash; personal atelier journal, not a storefront.

Voice: conversational and precise (“Find my scent”, “Mark worn”) — not corporate SaaS copy.

### Aesthetic Direction
**Resin Atelier** — editorial fragrance journal.

- Light-first atmosphere: cool stone / linen paper (not warm cream #F4F1EA), espresso ink with a faint olive undertone
- Accent: bottle-green resin, used sparingly for primary actions and focus
- Secondary warmth: champagne/brass for ranks and micro-highlights only
- Typography: expressive serif display (Cormorant Garamond) + geometric sans for UI (DM Sans) — never Inter/Roboto/system defaults as the brand face
- Shape: mostly soft-square (8–12px), not pill-heavy chrome
- Motion: short fade/rise on recommendations; respect `prefers-reduced-motion`
- Anti-references: purple SaaS gradients, neon-on-dark, glassmorphism, card-grid dashboards, terracotta-on-cream AI cliché, emoji decoration

### Implementation Defaults
- Framework: React + Vite + TypeScript (existing)
- Styling: vanilla CSS with CSS variables / design tokens in `src/index.css` (existing — preserve, do not introduce Tailwind unless explicitly requested)
- Component library: none (custom components in `src/components/`)
- Forms: custom controlled React forms (preserve)
- Data: client-only JSON seed + `localStorage` (preserve)
- Precedence: existing project stack wins

### Design Principles
1. **Feature first** — Today’s Pick (weather → occasion → mood → recommendations) is the hero workflow; chrome never outranks it.
2. **Hierarchy before color** — spacing, type scale, and weight establish order; resin green is accent, not wallpaper.
3. **One primary action per view** — e.g. “Find my scent” or “Mark worn today”; secondary actions stay quiet.
4. **Sensory density without clutter** — note pyramids and tags are scannable; avoid chip soup and stat strips in the first viewport.
5. **Personal tool honesty** — empty and never-worn states are useful, not decorative; no fake marketing claims.
